## 0. 공통 디자인 시스템 (즉시 적용 필요)

### 0-1. 컬러풀 아바타 컴포넌트
- 직원 이름 첫 글자 표시
- 배경색: 직원별 고정 랜덤 색상 (핑크 `#FF6B9D`, 파랑 `#4DABF7`, 노랑 `#FFD43B`, 초록 `#69DB7C`, 보라 `#9775FA`, 민트 `#38D9A9`, 오렌지 `#FF922B`)
- 사이즈: sm(24px), md(32px), lg(40px), xl(56px)
- 구현 방법: `userId` 또는 `name`을 해시해서 색상 인덱스 결정 → 항상 같은 색 유지

```tsx
// components/ui/Avatar.tsx
function getAvatarColor(name: string): string {
  const colors = ['#FF6B9D','#4DABF7','#FFD43B','#69DB7C','#9775FA','#38D9A9','#FF922B','#748FFC'];
  const index = name.charCodeAt(0) % colors.length;
  return colors[index];
}
```

### 0-2. 파스텔 뱃지/태그
- 현재: 단순 pill 뱃지
- 목표: 배경 파스텔 + 텍스트 진한 색, `rounded-full`, 폰트 `font-medium text-xs`
- 종류: 정규직(진회), 계약직(노랑), 소유자(보라), 관리자(초록), 의무(빨강 옅), 회사(파랑 옅), 개인(초록 옅)

### 0-3. 데이터 테이블 공통 패턴
- 헤더: `bg-gray-50`, `text-xs font-semibold text-gray-500 uppercase`
- 행 hover: `hover:bg-gray-50`
- 컬럼 정렬 아이콘 (오름/내림차순)
- 필터 드롭다운 (권한, 직위, 직책 등)
- 검색 인풋 (상단 좌측)
- 페이지네이션

### 0-4. 좌측 트리 사이드바 + 우측 콘텐츠 레이아웃
- 여러 페이지에서 공통으로 사용
- 좌측: 조직/팀 트리, 240px 고정
- 우측: 콘텐츠 영역, flex-1

## 1. 직원 관리 (`/team`) — 대폭 확장 필요

### 현재 구현
- 직원 목록 (이름, 이메일, 역할 정도)

### 추가 구현 필요

#### 1-1. 직원 목록 레이아웃 개선
- 좌측: 부서 트리 (부서명 + 인원수 뱃지)
- 우측: 직원 테이블
  - 컬럼: 아바타+이름, 직책/직위, 권한 뱃지 (소유자/관리자/직원), 부서
  - 상단 필터: 권한 필터, 직위 필터, 직책 필터
  - 검색: 이름으로 검색

#### 1-2. 직원 상세 프로필 페이지 (`/team/[id]`)
좌측 탭 메뉴:
- **기본정보** — 이름, 이메일, 주민등록번호(마스킹), 연락처, 주소
- **인사정보** — 입사일(재직기간 자동계산), 부서, 직위, 직책, 직무, 고용형태
- **경력** — 이전 회사명, 재직기간, 직위 (추가/수정 가능)
- **학력** — 학교명, 전공, 졸업여부, 기간
- **계약정보** — 근로계약서, 연봉계약서 등 (계약관리와 연결)
- **부가정보** — 첨부문서 (가족관계증명서, 주민등록등본 등 카테고리별 파일 업로드)
- **근무/휴기 설정**

우측 상단 액션:
- 퇴직신청 모달 (퇴직구분, 마지막근무일, 퇴직일, 퇴직사유 richtext)
- 휴직신청 모달 (휴직구분, 휴직기간, 휴직사유 richtext)
- 복직 처리 모달 (복직일, 이전 인사기록 유지/초기화)

#### 1-3. 인사노트 (`/team/notes`)
- 좌측: 직원 목록 (번호, 이름, 직책/직위, 부서, 노트건수)
- 우측: 선택된 직원의 노트 목록 + 새 노트 버튼
  - 노트 컬럼: 번호, 노트일, 노트구분, 제목, 작성자
  - 다운로드 아이콘

#### 1-4. 증명서 발급 (`/team/[id]/certificates`)
- 재직증명서, 경력증명서 카드 (발급횟수, 최근 발급일)
- 증명서발급신청 모달: 결재자 선택, 참조 선택, 증명서 종류, 용도
- 발급 미리보기: 성명, 주민번호, 주소, 부서/직위, 재직기간, 용도 표시

- 초과근무 시간: 초록색 텍스트

## 3. 급여 관리 (`/salary`) — 신규 모듈

### 현재 구현
- 없음

### 구현 필요

#### 3-1. 급여 목록 테이블
- 탭: 기본 인사정보 / 근무일 / 지급(비과세) / 지급(과세) / 공제 / 차인지급액 / 명세서
- 컬럼: 아바타+이름+직위, 근무일(%), 지급(비과세), 지급(과세), 공제, 차인지급액, 명세서 버튼
- 변동액 표시: `+300,000` (초록), `-141,126` (빨강)
- 우측 뱃지들: 정규직/계약직, 부양가족수, 건강보험/국민연금/고용보험 아이콘, 최저시급 경고
- `정산중`, `개별정산`, `계약필요` 상태 표시

#### 3-2. 급여명세서 상세 (`/salary/[id]/payslip`)
- 헤더: `N월 급여명세서`
- 실 지급액 (차인지급액) 크게 표시
- 지급 항목: 식비, 기본급, 연장근무수당, 휴일근무수당, 야간근무수당 등
- 공제 항목: 국민연금, 건강보험, 장기요양, 고용보험, 소득세 등
- 산출식 정보 섹션

#### 3-3. 백엔드 API
- `GET /salary` — 월별 급여 목록
- `GET /salary/:userId` — 직원별 급여 상세
- `GET /salary/:userId/payslip` — 명세서
- `POST /salary/calculate` — 급여 계산 (근무일 × 단가 + 수당 - 공제)

## 5. 전자결재 (`/approvals`) — 신규 모듈

### 현재 구현
- 없음 (collaboration 모듈은 있으나 결재 기능 없음)

### 구현 필요

#### 5-1. 결재함 목록 (`/approvals`)
- 검색바: 제목/문서분류로 검색
- 필터: 결재대기, 결재중, 반려, 결재완료
- 날짜 범위 필터
- 새 문서 버튼

- 테이블 컬럼: 제목+문서분류 뱃지, 결재상태, 작성일, 비고
- 문서분류 뱃지: 휴가(노랑), 연장근무(민트), 교육(파랑), 기안서(회색), 사유서(회색), 인사발령(보라)
- 결재상태: 결재하기+반려하기(대기), 결재대기/결재중/결재완료/읽음/읽지않음
- 비고: `결재해 주세요`, `결재자가 문서를 읽음(2/3)`, `결재(1/2)`, `참조문서 확인해 주세요`

#### 5-2. 결재 문서 상세
- 좌측: 문서 내용 + 첨부파일 뷰어 (PDF/문서 카드)
- 우측: 결재/반려/삭제 버튼, 결재현황(N/N), 참조(N/N), 댓글 입력+목록

#### 5-3. 새 문서 작성
- 문서 종류 선택 (휴가신청, 연장근무신청, 기안서, 사유서 등)
- 결재선 설정 (결재자, 참조자 다중 선택)
- Rich text 에디터
- 첨부파일 업로드

#### 5-4. 백엔드 API
- `GET /approvals` — 결재 목록 (내가 기안/내가 결재자)
- `POST /approvals` — 새 결재 문서 생성
- `POST /approvals/:id/approve` — 결재
- `POST /approvals/:id/reject` — 반려
- `POST /approvals/:id/comments` — 댓글
