## 1. 서비스 개요

**관리왕**은 중소사업장을 위한 올인원 직원 관리 SaaS 플랫폼입니다.

| 항목 | 내용 |
|------|------|
| 대상 | 5~300인 규모 중소·중견 기업 |
| 플랫폼 | 웹(Next.js 14) + 모바일(Expo React Native) |
| 배포 | Frontend: Vercel · Backend: Render · DB: PostgreSQL + Redis |
| 구조 | 멀티테넌트 (company_id 기반 Row-Level Isolation) |
| 인증 | JWT (Access 15분 + Refresh 7일) + Google OAuth |

## 2. 인증 & 계정 관리

### 2-1. 로그인 방식
- **이메일/비밀번호** 로그인
- **Google OAuth** 소셜 로그인
- 비밀번호 찾기 (이메일 링크 발송)
- 비밀번호 복잡도 검증 (8자↑, 영문+숫자+특수문자)

### 2-2. 직원 초대
| 방식 | 설명 |
|------|------|
| 이메일 초대 | 초대 메일 → 링크 클릭 → 비밀번호 설정 후 가입 |
| 공유 링크 초대 | 유효 기간(최대 30일) + 최대 사용 횟수 설정 가능 |
| SMS 초대 | 핸드폰 번호 직접 초대 (이름 사전 등록) |

### 2-3. 보안
- Refresh Token: HMAC-SHA256 해시로 DB 저장 (원문 저장 안 함)
- 로그인 Rate Limit: 5회/60초
- 이메일·이름: AES-256-GCM 암호화 저장 + HMAC 해시 인덱스

## 3. 직원 관리

### 3-1. 직원 프로필
- 기본 정보: 이름, 이메일, 전화번호, 프로필 이미지
- 인사 정보: 부서, 직책, 사번, 입사일
- 경력 사항: 이전 직장 경력 히스토리
- 학력 사항: 학교/전공/기간
- 첨부 문서: S3/R2 기반 파일 업로드 (이력서, 자격증 등)

### 3-2. 역할 체계
| 역할 | 설명 |
|------|------|
| `owner` | 회사 소유자. 모든 기능 접근 가능 |
| `manager` | 관리자. 담당 부서 범위 설정 가능 |
| `employee` | 일반 직원. 본인 정보만 접근 |

### 3-3. 팀 통계
- 전체/부서별/직책별 인원 현황
- 월별 입사자 수 (최근 12개월 트렌드)
- 활성/비활성 현황

## 4. 출퇴근 관리

### 4-1. 출퇴근 연동 방식 (다중 선택, 동시 활성화)

| 방식 | 설명 | 비고 |
|------|------|------|
| `manual` | 앱/웹에서 버튼 클릭 | 기본값 |
| `gps` | 회사 GPS 반경 내에서 처리 | GPS 설정 필요 |
| `wifi` | 사내 WiFi 연결 상태에서 처리 | SSID 사전 등록 필요 |
| `qr` | 관리자가 띄운 QR 스캔 | HMAC-SHA256 stateless 토큰, 자동 갱신 |
| `face` | 기기 지문/Face ID 인증 후 처리 | 모바일 v1.1 예정 |

**First-wins 정책**: 여러 방식이 동시에 활성화되어 있을 때, 먼저 처리된 방식이 기록됨.  
방식별 우선순위는 회사 `enabled` 배열 순서로 결정.

### 4-2. GPS 정책
- **Flag-not-Reject**: GPS 반경 밖이어도 출퇴근 허용, 플래그(`clock_in_out_of_range`)만 기록
- 정확도 보정: GPS 오차(accuracy)의 절반을 거리에서 차감 (유리한 방향)
- 엄격 모드(`gpsStrictMode`): 활성화 시 반경 밖 자동 거부 (선택적)

### 4-3. QR 출퇴근
- HMAC-SHA256 + timeWindow 기반 stateless 토큰
- Redis/DB 저장 없이 서버에서 즉시 검증
- 관리자 화면에서 QR 코드 표시 → 직원 스캔 → 자동 처리
- 갱신 주기: 1~60분 설정 가능 (기본 5분)
- 현재 + 이전 window 모두 허용 (시계 오차 대응)

### 4-4. 출퇴근 기록 항목
- 출근 시각, 퇴근 시각
- 사용 방식 (`clock_in_method`, `clock_out_method`)
- GPS 좌표, 거리, 반경 초과 여부
- 지각 여부, 지각 분수
- **실 근무시간** (총 근무 - 법정 휴게시간)
- **적용된 휴게시간** (법정 자동 또는 개인 설정)

### 4-5. 근태 현황 (관리자)
- 오늘 근태 현황 (전 직원 출퇴근 상태)
- 팀 통계 (지각, 결근, 조퇴 집계)
- 월간 뷰 (달력 형태)
- 근태 수동 수정 + 수정자/수정 시각 기록

### 4-6. 출퇴근 상태 분류
`pending` · `normal` · `late` · `early_leave` · `absent` · `half_day` · `vacation`

## 5. 개인 근무 스케줄

### 5-1. 개요
직원마다 계약서에 따라 출퇴근 시간이 다를 수 있습니다.  
관리왕은 **개인별 근무 스케줄**을 지원하며, 회사 기본값을 자동으로 fallback으로 사용합니다.

### 5-2. 스케줄 구성 요소

| 항목 | 설명 | 미설정 시 |
|------|------|----------|
| 출근 시간 | 개인 출근 기준 시각 (HH:mm) | 회사 설정 사용 |
| 퇴근 시간 | 개인 퇴근 기준 시각 (HH:mm) | 회사 설정 사용 |
| 휴게시간 | 일 휴게시간(분) | 법정 최소 자동 계산 |
| 지각 허용 | 지각 판정 허용 시간(분) | 회사 설정 사용 |

### 5-3. 법정 휴게시간 자동 계산 (근로기준법 제54조)

총 근무시간을 기준으로 법정 최소 휴게시간을 **자동 차감**합니다.

| 총 근무시간 | 법정 최소 휴게시간 |
|------------|------------------|
| 4시간 미만 | 없음 |
| 4시간 이상 | **30분** |
| 8시간 이상 | **60분** |

> 개인 휴게시간을 설정한 경우 법정 최소와 비교해 큰 값이 자동 적용됩니다.  
> **법정 기준 이하로는 설정 불가**합니다.

**실 근무시간 = 퇴근 시각 − 출근 시각 − 적용 휴게시간**

### 5-4. 스케줄 변경 권한 흐름

```
Owner / 계약 관리 권한자 → 즉시 변경 (PATCH /users/:id/work-schedule)
Manager 이하             → 결재 기안 생성 (POST /users/work-schedule-change)
                                    ↓
                          결재 완료 시 자동 적용
```

### 5-5. 탄력근무제 지원
- 직원별 개별 출퇴근 시간 설정으로 탄력적 근무제 운영 가능
- 출근 지각 판정 기준도 개인별 상이하게 설정 가능

## 6. 급여 관리

### 6-1. 기능
- 직원별 급여 기록 (기본급, 수당, 공제 등)
- 급여 명세서 발행
- 세금(원천징수) 자동 계산 참고

### 6-2. 접근 권한
**세분화된 급여 열람 권한 시스템**:

| 역할/권한 | 열람 범위 |
|-----------|----------|
| `owner` | 전체 직원 급여 |
| 재무/회계/경리 부서 관리자 | 전체 또는 담당 부서 (설정에 따라) |
| `canViewSalary` 권한 부여된 자 | 허용된 범위 내 |
| 일반 직원 | 본인 급여만 |

> 재무팀 키워드 자동 감지: 재무, 회계, finance, 경리, 급여

### 6-3. 권한 변경 절차
- Owner 또는 `canGrantSalaryAccess` 위임자 → 직접 변경
- 그 외 → 결재 기안(`permission_change`) 생성 → 승인 후 자동 적용

## 7. 휴가 & 결재

### 7-1. 휴가 관리
- 연차 신청, 반차, 특별휴가 등
- 잔여 연차 자동 계산
- 휴가 달력 뷰

### 7-2. 전자 결재 시스템

| 문서 유형 | 코드 | 설명 |
|----------|------|------|
| 일반 | `general` | 자유 형식 결재 문서 |
| 휴가 신청 | `vacation` | 휴가 신청서 |
| 지출 결의 | `expense` | 비용 처리 |
| 연장 근무 | `overtime` | 초과근무 승인 |
| 출장 | `business_trip` | 출장 신청 |
| 인사 발령 | `hr` | 인사 이동 |
| 권한 변경 | `permission_change` | 접근권한 변경 → **승인 시 자동 적용** |
| 근무 스케줄 변경 | `work_schedule_change` | 개인 근무시간 변경 → **승인 시 자동 적용** |

**결재 단계**: 다단계 결재선 지원 (step 1 → 2 → ... → 최종 승인)  
**자동 적용**: `permission_change`, `work_schedule_change` 문서는 최종 승인 시 시스템이 자동으로 변경사항을 적용합니다.

## 18. 접근 권한 시스템

### 18-1. 역할 기반 접근 제어 (RBAC)
`owner > manager > employee` 3단계 기본 권한

### 18-2. 세분화 권한 플래그

| 권한 | 설명 |
|------|------|
| `canInvite` | 직원 초대 권한 |
| `canManageContracts` | 계약서 관리 |
| `canManageEvaluations` | 평가 관리 |
| `canViewHrNotes` | HR 노트 열람 |
| `canManageHrNotes` | HR 노트 관리 |

### 18-3. 권한 변경 절차
- Owner / 위임자: 즉시 변경
- 그 외: 결재 기안 → 승인 후 자동 반영

### 18-4. 담당 부서 범위
Manager는 전체 또는 특정 부서만 담당하도록 설정 가능.  
`managedDepartments: null` = 전체 / `['개발팀', '기획팀']` = 해당 부서만

## 21. 마케팅 스튜디오

코드 배포 없이 랜딩 페이지 텍스트·배너·팝업을 관리할 수 있는 내부 CMS.

### 접속 방법
- URL: `<서비스 도메인>/marketing-studio`
- 인증: `MARKETING_STUDIO_KEY` 환경변수 값(스튜디오 키) 입력

### 기능

| 탭 | 기능 |
|----|------|
| 텍스트 블록 | 히어로/요금제/기능/CTA 섹션 문구 인라인 편집 + 일괄 저장 |
| 배너 | 상단 고정 띠 배너 CRUD (배경색·링크·기간·활성화) |
| 팝업 | 방문자 팝업 CRUD (즉시/딜레이/스크롤 트리거, 재표시 방지 N일) |

### 동작 방식
- 랜딩 페이지는 Next.js **Server Component** (ISR revalidate 60초)로 블록 값을 fetch
- 스튜디오에서 저장 즉시 반영 (최대 60초 딜레이)
- API 장애 시 코드에 하드코딩된 fallback 텍스트 자동 사용 → 랜딩 절대 깨지지 않음

### 배포 전 필수 작업
1. Render 환경변수 `MARKETING_STUDIO_KEY` 설정 (임의 문자열, 최소 8자)
2. DB 마이그레이션 `1746500000000-CreateMarketingTables` 실행
3. (선택) Vercel 환경변수 `INTERNAL_API_URL` 설정 → SSR 내부 API 호출 최적화
